Create a new repository

  1. Repositories -> New -> 'make name our Repository' + Initialize this Repository with a README -> push the button 'Create Repository'
  2. Our need link for this Repository
  3. push 'Clone or download' and copy this link
  4. Переходим на PC в нашу папку с проектами и открываем там GitBash Here -> вписываю команду code, что откроет мне VSCode в данной папке -> в терминале VSCode выполняем команду git clone 'наш линк с GitHub' -> открываем в VSCode нашу новозагрузившуюся папку 'my-project-name'

  • git status - check if there is a repository in this folder
  • git init -настроит систему котроля версий на работу с данной папкой
  • ls -a - команда покажет все файлы находящиеся в этой директории

    • Command for save and push on GitHub

  • git add . - данная команда даст возможность подготовить к сохранению все файлы "." находящиеся в текущей директории git add index.html - команда подготавливает файл index.html к сохранению
  • git reset index.html - команда даст возможность не сохранять изменения в файле index.html, на случай, если я передумал
  • git commit -m 'This write changed was been' - эта команда выполняет сохранение
  • git add index.html + git commit -m == git commit -am
  • git push origin develop

    • Other commands git

  • git log - команда позволяющая посмотреть историю всех сохранений
  • git checkout ab72d91799d74e4212ac35afcf380f5ca68b1804 - переход по номеру сохранённого коммита - возврат к определённому сохранённому коммиту
  • git checkout master - переход к ветке master - возврат к определённому сохранённому коммиту
  • git branch develop - создаём новую ветку с названием 'develop'
  • git checkout develop
  • git branch develop + git checkout develop == git checkout -b new-branch-name
  • git branch - список всех доступных веток
  • git branch -D new-branch-name - удалит ветку с именем new-branch-name, только необходимо перед этим перейти с неё на другую ветку
  • git remote - команда покажет с каким сервером на GitHub связан проект
  • git remote show origin - команда покажет какой именно сервер прячется за словом origin
  • git remote remove origin - команда удалит сервер на GitHub связан проект
  • git remote add origin 'та самая ссылка с GitHub'- команда добавит сервер на GitHub связан проект
  • git pull origin develop - скачает все произошедшие изменения с GitHub
